## 1. Introduction

In this project, we developed a web-based QR ordering system called QuickPay, designed for restaurants and cafes. The system allows customers to scan a QR code, access the menu, place orders, and split the bill without interacting with a waiter.

The system also includes an AI-based recommendation feature that suggests meals based on user preferences such as dietary restrictions, or just personal preferences  like gluten intolerance. Additionally, the split bill functionality allows multiple users to pay only for their own items, improving efficiency and user experience.

My role focused on designing and implementing a modern, responsive, and user-friendly frontend interface that interacts seamlessly with the backend system.

## 4. Work I Have Done

### 4.1 UI/UX Design for QR Ordering System

I designed a clean  interface where users can:

- Scan a QR code and access the menu  
- Browse food and drink options  
- Add items to their order  

### 4.2 Responsive Mobile-First Design

Since the application is primarily used on mobile devices, I implemented a mobile-first design using Tailwind CSS to ensure compatibility across different screen sizes.

### 4.3 Component-Based Development (React)

I developed reusable components such as:

- MenuCard → displays menu items  
- OrderSummary → shows selected items  
- PaymentPanel → handles payment options  

This approach improved maintainability and scalability.

### 4.4 State Management and User Interaction

I used React hooks (useState) to manage:

- Selected menu items  
- Cart updates  
- Payment choices  

User actions such as adding/removing items and confirming orders were handled using event handling in React.

### 4.5 Frontend–Backend Integration

I connected the frontend with the FastAPI backend using HTTP requests:

- GET requests to fetch menu data  
- POST requests to send order data  

The data was handled in JSON format to ensure smooth communication between client and server.

### 4.6 Payment and Split Bill Interface

I designed a user-friendly payment interface where users can:

- Choose payment methods (card or cash)  
- Split the bill based on selected items, or even between groups  

This required careful UX design to avoid confusion and ensure clarity.

### 4.7 AI Recommendation Interface

I created the UI for the recommendation system where users can:

- Enter dietary or personal preferences  
- Receive meal suggestions from an external API  

The results are dynamically displayed on the interface.
